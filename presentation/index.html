Domain Driven Design
c'est quoi ?

  1. Une méthode de conception logicielle qui favorise une conception proche de la réalité
  2. Focalisation sur le métier (au lieu d’uniquement des parties techniques)
  3. On utilise explicitement des noms et des verbes du domaine dans le code
  4. La connaissance métier est le noyau de l’application

Langage ubiquitaire

Contextes bornés (Bounded Context)

  1. Un Bounded Context est un contexte délimité par une frontière linguistique nette.
  2. Dans un Bounded Context, le modèle du domaine est exprimé par un Ubiquitous Language
  3. Un domaine métier peut contenir plusieurs Bounded Context
  4. chaque Bounded Context définissant son Propre Language

Contextes bornés (Bounded Context): Avantages

  1. Renforce la pureté et la puissance du modèle
  2. Evite la confusion avec les autres contextes
  3. Simplifie l’Architecture (Fait émerger des transitions explicites entre les différents contextes)
  4. Favorise un partitionnement technique et organisationnel

Context Mapping

  1. Permet d'identifier les points de contact et les relations entre les différents modèles
  2. On identifie les Bounded Context amont et aval
  3. On analyse les transformations nécessaires
  4. ça peut être une relation entre deux équipes
  5. L’équipe en amont (upstream)
  6. L’équipe en aval (downstream)

Domain Events

  1. modélisent l'activité dans le domaine par une série d'événements discrets
  2. S'est produit dans le passé
  3. Intéresse les experts métier
  4. Fait partie de l'Ubiquitous language

Contenu Domain Events

  1. IDENTIFIANTS des entités impactées
  2. DATE/HEURE de survenance
  3. Le CONTEXTE avant et après l'événement
  4. NUMERO DE SERIE de l'événement
  5. PAS DE COMPORTEMENT !!!!

Value Object

  1. Une valeur est intangible et immuable
  2. Quantité ou description immuable
  3. ensemble de valeurs avec un nom explicitant le sens de cet ensemble
  4. un comportement propre au nouveau concept

Contenu Value Object

  1. MAGNITUDE et UNITE Conversion vers une autre unité
  2. CONVERSION depuis/vers représentation textuelle
  3. REGLES DE VALIDATION Appartenance à un intervalle, conformité à un format, ..
  4. COMPORTEMENT propre au domaine
  5. PAS D'IDENTITE !!!!

Entité

  1. IDENTITE Unique et Immutable
  2. ETAT mutable à travers le temps
  3. RELATIONS avec d'autres Entités (associations)
  4. COMPORTEMENT METIER utilisant l'état encapsulé

Agrégats

  1. Un et un seul AGGREGATE ROOT point d'entrée unique de l'agrégat
  2. Des ENTITES Quelques unes au maximum
  3. Des VALUE OBJECTS sans limitation de nombre
  4. Un (ou plusieurs) invariant

Racine d'agrégats

  1. Des VALUE OBJECTS Identités
  2. Des VALUE OBJECTS caractérisant son état
  3. Les COMMANDES de modification de l'Agrégat
  4. Les QUERIES de calcul d'un état agrégé
  5. Des références vers des ENTITES Du même Agrégat

repositories

  1. permet de charger l'Aggregate Root
  2. encapsule la création, la suppression et la modification d'un Agrégat
  3. Les Repositories représentent un moyen d'accès à un ensemble d'objets
  4. Ils donnent l'illusion d'une simple collection en mémoire

factories

  1. permet la création des Agrégats complexe
  2. produit un Agrégat complètement construit, satisfaisant tous ses invariants.
  3. représente une création métier d'un agrégat

Services

Application Service

  1. Chaque use case va s’incarner par un Application Service
  2. Utilisation d’une forme verbale (Verbe + Complément d’objet) Notion d’Action
  3. La transaction métier est au niveau du use case
  4. On ne peux composer les Application Service entre eux
  5. Pas d’état

Domain Service

  1. Comporte des règles métier
  2. Implémente un concept du domaine qu'il n'est pas naturel à une Entité ou à un Value Object
  3. Exemple: Comportement à cheval sur plusieurs Entités
  4. Composition possible d'un Domain Service à partir d'autres Domain Services

Infrastructure Service

  1. Il contient des éléments techniques Exemples: Intégration, Communication extérieure
  2. L’interface est dans la couche Domain
  3. L’implémentation est dans la couche Infrastructure
